# MyPath.Care

MyPath.Care is a care coordination platform built for healthcare, with an initial focus on workers' compensation. It connects patients, providers, case managers, adjusters, and employers around a single patient's care journey — replacing the phone-tag, fax-chains, and parallel-record-keeping that fragment care today.

## Platforms

- **API** — Go service powering every client with a unified data contract.
- **iOS** — Native SwiftUI app for iPhone and iPad, built around modern iOS 26 design idioms.
- **Android** — Native Jetpack Compose + Kotlin app for phone and tablet, built around Material 3 Expressive design.
- **Web** — Desktop-first React + shadcn app with a clean, modern design language built around clarity, generous whitespace, and refined typography.

## How We Build

MyPath.Care is engineered by humans collaborating with AI agents across every stage of the software development lifecycle. The process is deliberately structured — not because the agents need rails, but because **rigor at the artifact level is what makes AI-accelerated development trustworthy at production scale**.

Every feature begins with a specification co-authored across every layer of the platform — API, iOS, Android, and Web — before a line of code is written. Implementation then lands in parallel across every target in hours of agent time, not weeks of human-team time. Founders iterate against the working prototype until the product is right, with agents working in parallel across every platform. Every change leaves behind a complete audit trail: what we set out to build, what changed during iteration, what shipped, and what now exists in the product.

Cross-platform parity is functional, not visual: each platform feels native to its idiom, but every behavior does the same thing on every client. The result is a development pipeline that ships coherent, multi-platform features at the velocity of a small team — without the documentation debt or platform drift that usually come with moving fast.
